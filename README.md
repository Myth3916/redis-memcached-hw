## Домашнее задание к занятию "Кеширование Redis/memcached" - Шаров Олег

### Задание 1  
Приведите примеры проблем, которые может решить кеширование.

Кеширование позволяет решить следующие типичные проблемы:

1. **Высокая нагрузка на источник данных** — например, частые запросы к базе данных или внешнему API могут перегружать систему. Кеширование уменьшает количество обращений к источнику, возвращая ранее полученные данные из быстрого промежуточного хранилища.

2. **Медленное время отклика приложения** — если каждый запрос требует выполнения сложных вычислений или обращения к медленным ресурсам (например, дисковой БД), пользователь сталкивается с задержками. Кеш хранит результаты таких операций в оперативной памяти, обеспечивая почти мгновенный доступ.

3. **Повторяющиеся запросы от множества пользователей** — например, главная страница интернет-магазина запрашивается тысячами пользователей. Без кеширования сервер каждый раз генерирует один и тот же контент. С кешем — генерация происходит один раз, остальные получают готовый ответ.

4. **Ограничения пропускной способности или лимиты API** — внешние сервисы часто ограничивают число запросов в минуту/час. Кеширование позволяет соблюдать эти лимиты, повторно используя уже полученные данные.

5. **Снижение затрат на инфраструктуру** — за счёт уменьшения числа обращений к «тяжёлым» компонентам (БД, микросервисам) можно обойтись меньшим количеством серверов или более слабыми конфигурациями.

6. **Обеспечение отказоустойчивости при кратковременных сбоях** — если источник данных временно недоступен, кеш может продолжать отдавать актуальные (или хотя бы устаревшие, но приемлемые) данные, сохраняя работоспособность приложения.
